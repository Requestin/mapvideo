## Зафиксированные архитектурные решения (не обсуждаются повторно)
- Рендер видео: ТОЛЬКО серверный (Puppeteer + FFmpeg). MediaRecorder не используем.
- Кадры Puppeteer идут в stdin FFmpeg через image2pipe, НЕ пишутся как PNG на диск.
- Очередь рендеров хранится в PostgreSQL (единая таблица `render_jobs`, отдельной `videos` нет). Переживает рестарт. Single-worker, до 5 в очереди.
- История видео = `render_jobs WHERE status='done'` (отдельной таблицы `videos` не существует).
- Для внутреннего доступа Puppeteer к `GET /api/render/state/:jobId` используется одноразовый `render_token` (hash-only хранение, TTL, привязка к job).
- Авторизация: httpOnly session cookie (opaque токен, в БД sha256-хэш) + double-submit CSRF (csrf_token cookie + X-CSRF-Token заголовок). JWT и SESSION_SECRET не используем.
- CSRF требуется на все мутации КРОМЕ `POST /api/auth/login` (защищён SameSite=Lax + rate-limit).
- Rate limit на POST /api/auth/login: 5 неудачных попыток с IP за 10 мин.
- Имена файлов, директорий, таблиц БД, колонок, JSON-ключей API — английский. Тексты ошибок и логи — русский.
- PixiJS pinned в 7.4.2 (не v8). GSAP без премиум плагинов — движение по пути пишем сами.
- OSM по умолчанию: russia-260419.osm.pbf (уже лежит в osm-data/). Переключение на planet — одна переменная OSM_PBF_FILE в .env.
- Сеть: наружу открыты ТОЛЬКО 80/443 через хостовой nginx, docker-nginx запрещен.

## Порядок работы
1. Открой SPEC.md, найди фазу со статусом "🔄 В работе" (если нет — начни с первой "Не начато").
2. Открой соответствующий taskN.md. Найди раздел "## Заметка для следующей сессии" — если там что-то есть, продолжай оттуда.
3. Возьми ПЕРВУЮ незакрытую задачу (- [ ]) из раздела "## Задачи".
4. Отметь её [~] в taskN.md и поставь фазу в "🔄 В работе" в SPEC.md (если ещё нет).
5. Реализуй ТОЛЬКО эту задачу. Минимум кода, никаких "улучшений" соседнего кода.
6. Проверь: компилируется (`npm run typecheck`), тесты зелёные (`npm test`), линтер не ругается или прогони релевантные проверки для измененной области.
7. Отметь задачу [x]. Обнови "## Заметка для следующей сессии" в taskN.md.
8. Если вся фаза завершена — поставь "✅ Сделано" в SPEC.md и переходи к следующей. Если нет — остановись и жди команды.

## Stop conditions (ОСТАНОВИСЬ и спроси)
- Нужно удалить файл, дропнуть таблицу, сделать git reset --hard, git push --force
- Нужно добавить новую зависимость в package.json
- Нужно тронуть nginx конфиг maps.gyhyry.ru или любой файл на хосте вне проекта
- Нашлась несостыковка между task-файлом и SPEC.md/cursor.md
- Поведение требует решения, не зафиксированного в таске (UX/визуал/логика)
- Задача кажется "подозрительно большой" — предложи разбить

## Forbidden
- Не трогать /etc/nginx/ на хосте (независимый проект maps.gyhyry.ru)
- Не коммитить .env, osm-data/, data/videos/
- Не использовать JWT, MediaRecorder, GSAP MotionPath, PixiJS v8 async API
- Не создавать отдельную таблицу `videos` и не возвращать в API `userId` из клиентского state (брать всегда из `req.user.id`)
- Не писать логи на английском
- Не именовать таблицы/колонки/файлы ассетов на русском
- Не запускать destructive git команды без явного разрешения
- Не закрывать задачу как [x] пока не прошли typecheck и тесты
