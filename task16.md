## Задачи

### 16.1 — Вернуть inline-ошибку на форму логина (п.5 ревью)

**Файл:** `frontend/src/pages/login-page.tsx`

**Проблема:** Ошибка входа показывается только в toast справа снизу, который пропадает через 6 сек. Пользователь может не успеть прочитать.

**Решение:** Вернуть `error` state и отображение под полями формы **в дополнение** к toast. Toast остаётся для консистентности, inline — чтобы ошибка не потерялась.

### 16.2 — Исправить текст toast «скачано» при отсутствии downloadUrl (п.6 ревью)

**Файл:** `frontend/src/components/editor-workspace.tsx`

**Проблема:** Сообщение «Видео успешно создано и скачано» показывается всегда, даже если `downloadUrl` пуст и файл не скачан.

**Решение:**

```typescript
if (s.downloadUrl) {
  a.href = s.downloadUrl; a.click();
  showToast({ type: 'success', message: 'Видео успешно создано и скачано.' });
} else {
  showToast({ type: 'success', message: 'Видео создано. Скачайте из «Моя история».' });
}
```

### 16.3 — Показывать ошибку при сбое поллинга рендера (п.7 ревью)

**Файл:** `frontend/src/components/editor-workspace.tsx`

**Проблема:** Если при опросе статуса рендера пропала сеть — `catch` пуст, прогресс-бар зависает.

**Решение:** Считать подряд неудачные попытки. После 5 сбоев показать toast с предупреждением. После 15 — отменить поллинг и сбросить состояние.

```typescript
} catch {
  networkFailCount++;
  if (networkFailCount === 5) {
    showToast({ type: 'warning', message: 'Потеряна связь с сервером. Проверяем…' });
  }
  if (networkFailCount >= 15) {
    setRenderJobId(null);
    showToast({ type: 'error', message: 'Связь потеряна. Проверьте рендер в «Моя история».' });
  }
}
```

### 16.4 — Привести admin-page к единому стилю ошибок (п.9 ревью)

**Файл:** `frontend/src/pages/admin-page.tsx`

**Проблема:** Загрузка списка и создание пользователя используют inline-ошибки, а удаление — toast. Непоследовательно.

**Решение:** Оставить inline для **формы создания** (логично — ошибка привязана к форме). Для **загрузки списка** — показывать toast + кнопку «Повторить». Для **удаления** — toast (уже есть). Документировать стратегию: «форма — inline + toast, фоновые операции — только toast».
