## Заметка для следующей сессии

### Геокод (13.1)

Два источника проблемы, оба исправлены:

1. **Дубликаты.** Photon на запрос «Мос» возвращал 4 × «Москва, Россия» (разные
   OSM-объекты, один и тот же `name + city + country`). Дедупликация — на
   бэке в `backend/src/routes/geocode.ts`:
   - оверфетчим Photon в 3 раза от запрошенного `limit` (до потолка 20), чтобы
     после схлопывания дублей у пользователя осталось достаточно уникальных
     вариантов;
   - ключ группировки — `fullName.trim().toLowerCase()`, первый встретившийся
     кандидат побеждает (Photon уже ранжирует по релевантности);
   - пустые `fullName` не трогаем — их отфильтрует фронт.
2. **«Двойное открытие».** После `pick(r)` модалка писала `r.fullName` в
   `query`, и дебаунс-эффект тут же повторно фетчил ту же строку — саджест
   появлялся второй раз поверх только что выбранного варианта. Починено
   guard-ом в `add-point-modal.tsx`: если `picked && picked.fullName === query`,
   то эффект не запускает запрос (и очищает `results`/`loading`/`error`).
   `picked` добавлен в deps эффекта, чтобы при редактировании текста
   (когда `setPicked(null)` срабатывает) снова включался живой поиск.

### Начертания шрифтов (13.3)

Корневая причина «виден только Montserrat и только один вариант» — старый
`@font-face` инжектился без дескриптора `font-weight`, поэтому браузер
регистрировал все файлы `Montserrat-*.ttf` как один и тот же вариант
(Regular-400) и перетирал предыдущие. PIXI.Text дальше просто рендерил то,
что пришло последним, а все настройки `fontWeight` игнорировались.

Изменения:

- **Backend (`backend/src/routes/fonts.ts`).** `FontEntry` теперь содержит
  `weight: number` (CSS 100..900) и `weightLabel: string` (локализованное
  имя начертания). Имя файла парсится как `Family-Suffix.ext`; суффикс
  мапится через словарь `WEIGHT_BY_NAME` (Thin=100, ExtraLight=200 …
  Bold=700, Black=900; Regular/Book/Normal=400). Файл без суффикса
  (`Supermolot.ttf`) трактуется как Regular-400. Неизвестные суффиксы
  получают `weight=400` и свой оригинальный суффикс в `weightLabel` —
  лучше, чем терять файл.
- **Frontend API (`frontend/src/api/fonts.ts`).** `FontEntry` расширена
  теми же полями.
- **`frontend/src/services/fonts.ts`.** `@font-face` теперь включает
  `font-weight: ${f.weight}`, а `document.fonts.load()` вызывается с
  `${weight} 16px "<family>"`, иначе браузер лениво оставит только
  Regular и PIXI увидит неправильный файл при первом рендере.
- **`LabelSettings`.** В `frontend/src/state/types.ts` добавлено
  `fontWeight: number`, `defaultLabelSettings()` выставляет 400. Старые
  подписи (у которых поля ещё нет) читаются с fallback на 400 в рендерере
  и в панели, так что миграция не требуется.
- **`frontend/src/pixi/label-renderer.ts`.** В `PIXI.TextStyle` пробрасываем
  `fontWeight: String(settings.fontWeight ?? 400) as PIXI.TextStyleFontWeight`
  — PIXI принимает строковые значения `'400'`/`'700'` и согласует их с
  `@font-face`.
- **`frontend/src/components/right-sidebar/label-settings-panel.tsx`.**
  Вместо массива `families: string[]` держим полный `fonts: FontEntry[]`.
  Селект «Семейство» рендерится из уникальных `family`, новый селект
  «Начертание» — из доступных для текущего семейства вариантов
  (`weightsForFamily`, дедупликация по `weight`). При смене семейства
  сохраняем текущий вес, если он есть у нового шрифта, иначе съезжаем на
  400 или ближайший доступный — так пользователь не увидит беззвучной
  подмены стиля.
